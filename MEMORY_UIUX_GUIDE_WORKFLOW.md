# UI/UX Guide Workflow Memory

## 핵심 계획

현재 `wireframe` 페이지는 실제 서비스가 아니라, 이후 개발 시 참조할 UI/UX 기준 샘플북으로 사용한다.

나중에 다른 화면이나 프로젝트를 만들 때 사용자는 다음처럼 지시할 수 있다.

```md
wireframe의 Pattern 02 기준으로 만들어줘.
Grid Rule, Dock Rule, Form Rule은 기존 기준 그대로 적용해줘.
```

## 기준화 방식

1. 먼저 화면 패턴을 실제 동작하는 와이어프레임으로 만든다.
2. 사용자가 테스트하면서 UI/UX 규칙을 확정한다.
3. 확정된 패턴과 규칙을 `UIUX_GUIDE.md` 같은 마크다운 문서로 정리한다.
4. 이후 다른 프로젝트에서 해당 마크다운 파일 경로를 알려주면, Codex가 그 파일을 읽고 기준으로 삼아 개발한다.

## 추천 파일

- `UIUX_GUIDE.md`: 패턴과 규칙을 정리하는 기준 문서
- `wireframe`: 눈으로 확인하는 동작 샘플

## 패턴 예시

- `Pattern 01: 데이터 그리드`
- `Pattern 02: 카드&그리드`
- `Pattern 03: 카드&입력패널`
- `Pattern 04: 그리드&하단 탭`

## 공통 규칙 예시

- `Grid Rule`: 정렬, 행 선택, 체크박스, 데이터 정렬 방식
- `Form Rule`: 필수 입력폼 2개, 노란 배경, 붉은 `*`
- `Dock Rule`: 행 선택/체크 선택에 따른 하단 도크 패널 동작
- `Tab Rule`: 탭 표현 방식과 활성 탭 스타일

## 중요한 약속

- 선택과 체크는 서로 다른 상태다.
- 한쪽 상태가 있는 상태에서 다른 상태를 수행하려 하면 경고창으로 확인한다.
- 경고창에서 `확인`은 항상 사용자가 방금 하려던 최신 작업을 수행한다.
- `취소`는 기존 상태를 유지한다.

## 다음에 이어갈 때

```md
지금 만든 wireframe 기준을 `UIUX_GUIDE.md`로 정리해두면,

```
